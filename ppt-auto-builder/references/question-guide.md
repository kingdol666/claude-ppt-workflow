# 提问指南

> 阶段一：用 `AskUserQuestion` 一次性发出问题。根据场景选用合适的模板。

---

## 核心原则

- **一次问完**：不要分多轮，所有缺失信息用一组 `AskUserQuestion` 获取
- **智能推荐**：根据主题推断受众、页数、风格，减少提问量
- **Full Auto 模式**：用户已明确指定参数时，跳过提问直接进入知识收集

---

## 场景模板

### 场景A：学术研究（论文答辩、开题报告、研究汇报）

```
Q1: 研究主题是什么？有论文/报告/数据文件提供吗？
Q2: 页数：8页（精简）/ 10页（标准答辩）/ 12页（详细）/ 15页+（综合汇报）
Q3: 风格推荐：Study Style #20 双模学术（暖+锐）★、Fathom数据叙事(P2)、工程蓝图(#14)
Q4: 受众：答辩委员会 / 同行学者 / 跨学科听众
Q5: 模式：Guided（关键节点确认）/ Full Auto（一路到底）
```

## 智能推断规则

当用户只给了简短描述（如"做个碳化硅PPT"）时，按以下规则推断：

**场景冲突解决**：当多个关键词匹配不同场景时，选择与"格式"（答辩/汇报/路演）最具体的场景，而非内容领域。例如"碳化硅学术答辩PPT" → 选择场景A（学术答辩）而非场景H（材料/工程）。

推断值应作为预选默认值在 AskUserQuestion 选项中展示，用户可覆盖。仅当用户在消息中明确指定值时才跳过该问题。

| 关键词 | 推断结果 |
|--------|---------|
| 论文、答辩、开题、研究 | 学术场景 → Study Style #20 双模学术，10页，Guided |
| 汇报、报告、总结、规划、复盘 | 商务场景 → NYT Magazine，10页，Guided |
| 路演、发布、Pitch、Demo | 路演场景 → 苏联构成主义，10页，Guided |
| 市场、行业、趋势、竞争、分析 | 行业分析 → Pentagram编辑，12页，Guided |
| 培训、教学、课件、课程、教程 | 教学场景 → Neo-Brutalism，12页，Guided |
| 技术、架构、代码、开发、AI | 技术场景 → xkcd白板，8页，Guided |
| 科普、介绍、入门、了解、什么是 | 科普场景 → Neo-Pop，8页，Guided |
| 材料、工程、制造、工艺 | 材料场景 → Study Style #20 双模学术，8页，Guided |

## 已明确参数检测

提问题前先检查用户消息中是否已包含以下信息，已明确的跳过不问：
- 主题/标题 → 跳过Q1
- 页数 → 跳过Q2
- 风格 → 跳过Q3
- 受众 → 跳过Q4
- "直接做"/"一路到底"/"Full Auto" → Q5设为Full Auto
