# HTML 幻灯片视觉审查指南

> 用于 ppt-auto-builder 的三轮审查闭环：
> - **第一轮（阶段四 4.2）**：排版审查 — 截图 + 4 维打分（排版/层次/内容/风格），图文协调暂不评
> - **第二轮（阶段五 5.3）**：图文审查 — 仅评图文协调维度，诊断图片问题并决定是否重新生图
> - **第三轮（阶段六 6.3）**：最终抽检 — 封面 + 配图页 + 深度页的整体效果确认

---

## 1. 截图生成

### Playwright 截图（首选）

逐个 slide 截图，使用 Windows 绝对路径（避免 cygpath 依赖）：

```bash
npx playwright screenshot --wait-for-timeout=1000 \
  "file:///C:/Users/87287/workspace/ppt_workspace/slides/slide-01.html" \
  "C:/Users/87287/workspace/ppt_workspace/slides/_review_01.png"
```

对应每个 slide-XX.html 生成 _review_XX.png。如果 slide 数量多，用 PowerShell：

```powershell
Get-ChildItem ppt_workspace/slides/slide-*.html | ForEach-Object {
  $n = $_.BaseName -replace 'slide-',''
  npx playwright screenshot --wait-for-timeout=1000 `
    "file:///$($_.FullName -replace '\\','/')" `
    "ppt_workspace/slides/_review_$n.png"
}
```

### CDP 截图（备选）

如果 playwright screenshot 不可用，使用 web-access skill 的 CDP 截图接口。

### 降级方案

如果截图完全不可行，改用 Read 工具直接阅读 HTML 源码，通过代码审查判断布局结构。

## 2. 审查维度与打分标准

每页按 5 个维度打分（1-5），总分 25 分。

### 维度 1：排版与布局（Layout）

| 分数 | 标准 |
|------|------|
| 5 | 布局完美匹配论证逻辑，与前页截然不同，密度呼吸感明确 |
| 4 | 布局匹配论证，与前页有区别，密度基本合理 |
| 3 | 布局合理但与前页雷同，或密度选择可商榷 |
| 2 | 布局与论证逻辑不匹配（如对比型内容用了罗列式布局），或连续3页同布局 |
| 1 | 布局混乱，无逻辑可循 |

**审查要点**：相邻页是否不同？布局是否服务于论证？密度是否有呼吸感？

### 维度 2：图文协调（Image-Text Harmony）

| 分数 | 标准 |
|------|------|
| 5 | 配图尺寸/位置精准，图说80%故事，文字做标注；text-only页排版即为视觉设计 |
| 4 | 图文比例合理，图有信息量，文字不淹没图 |
| 3 | 有图但偏小/偏大，或图与文字各说各的 |
| 2 | 图片被文字淹没（尺寸<20%面积），或图与内容脱节，或该有图却没有 |
| 1 | 配图严重干扰阅读，或风格/内容完全不搭 |

**审查要点**：
- 配图页：图片占50%+面积？文字只做标注？图在说和文相关的事吗？
- text-only页：色块/卡片/数字大字是否代替了图的视觉作用？

### 维度 3：视觉层次（Visual Hierarchy）

| 分数 | 标准 |
|------|------|
| 5 | 标题→正文→脚注层次清晰，关键数据/图表突出，视觉锚点3秒内可见 |
| 4 | 层次分明，主次清楚 |
| 3 | 层次存在但不够强（标题和正文差距≤4pt字号） |
| 2 | 所有元素同等大小，缺乏层次引导 |
| 1 | 混乱层级，读者不知道先看哪 |

**审查要点**：字号阶梯是否≥6pt？关键数据是否有视觉突出（如超大数字、色块高亮）？

### 维度 4：内容质量（Content Quality）

| 分数 | 标准 |
|------|------|
| 5 | 数据有出处和上下文解释，论点有逻辑递进，引用了 references/（知识库目录） 的具体数据 |
| 4 | 内容充实，有数据和引用 |
| 3 | 内容正确但偏干（数据裸奔无解释），或偏水（空洞无数据） |
| 2 | 要点式碎片，缺上下文和解释 |
| 1 | 内容错误、编造引用、或与主题无关 |

**审查要点**：数据有出处吗？有逻辑递进吗？深度页展开了吗？极简页精炼了吗？

### 维度 5：风格一致（Style Consistency）

| 分数 | 标准 |
|------|------|
| 5 | 签名元素（页眉/底部栏/配色/字体）全篇统一，风格特征鲜明 |
| 4 | 全篇一致，偶有小偏差（如字号差1pt） |
| 3 | 基本一致，但某页卡片色/边框样式有偏离 |
| 2 | 某页明显偏离风格系统（换了配色/字体） |
| 1 | 多页风格不一致，像不同PPT拼起来的 |

**审查要点**：配色是否全篇统一？字体族是否一致？签名元素是否每页出现？

## 3. 诊断与分类

```
每页总分 ≥ 20分（5项×4分） → ✅ PASS
每页总分 15-19分           → ⚠️ MINOR（小幅修改：字号/间距/颜色调整）
每页总分 < 15分            → ❌ FAIL（需要重做该页）
```

**第一轮排版审查（4维适配）**：本轮跳过"图文协调"维度，按 4维20分评判。
- PASS ≥16 / MINOR 12-15 / FAIL <12
- "排版与布局" ≤2 → 无论总分 → 重做该页

**关键规则（触发重做）**：
- "排版与布局" ≤ 2 分 → 无论总分 → 重做该页
- "图文协调" ≤ 2 分 → 触发**生图诊断**（见第4节）
- "内容质量" ≤ 2 分 → 重新确认 references/（知识库目录） 数据 → 重写该页

## 4. 生图决策（Image Regeneration Decision）

> **当"图文协调"评分 ≤ 2 时触发。**

### 诊断流程

```
图文协调 ≤ 2 → 判断问题根因：

├── 图片内容与页面论点不相关或弱相关？
│   → 重新写 Prompt（强化内容锚点，从该页 PLAN.md 要点提取核心视觉元素）
│   → 回阶段四：用 gpt-image-2 重新生成该图
│   → 占位符 id 不变，新图覆盖旧文件
│
├── 图片尺寸/比例与布局模式不匹配？
│   → 调整 HTML 占位符 CSS 区域（w/h）
│   → 按新尺寸重新生成图片（传正确比例参数，如 --size 1024x1024）
│   → 如仅需裁切，也可用 sharp 等工具处理
│
├── 图片视觉风格与 PPT 整体风格割裂？
│   → Prompt 中强化风格关键词（从 PLAN.md "风格参数速查" 提取）
│   → 重新生成该图
│
├── 本页应该配图但被错误标记为 text-only？
│   → 新写配图 Prompt（三要素：内容锚点+风格关键词+布局约束）
│   → 阶段四生图
│   → HTML 添加占位符 div（选匹配的布局模式）
│   → 通知 huashu-slides "修复 slide-0X：添加配图"重建该页
│
└── 本页不该配图但放了图（装饰性图片）？
    → 移除占位符 div，改为 text-only 排版
    → 通知 huashu-slides "修复 slide-0X：移除配图"重建该页
```

### 重新生图后的验证

生图完成后必须：
1. 确认新图文件已写入 `garden-gpt-image-2/image/`
2. 重新截图该页
3. Read 截图确认图文协调 ≥ 3 分
4. 更新审查报告

## 5. 审查报告模板

```markdown
## 视觉审查报告

### 总览

| 指标 | 数值 |
|------|------|
| 审查页数 | N |
| ✅ PASS（≥20分） | X 页 |
| ⚠️ MINOR（15-19分） | Y 页 |
| ❌ FAIL（<15分） | Z 页 |
| 生图决策触发 | K 页 |

### 逐页评分

| 页 | 标题 | 排版 | 图文 | 层次 | 内容 | 风格 | 总分 | 判定 |
|----|------|------|------|------|------|------|------|------|
| 1 | 封面 | 5 | 5 | 4 | 4 | 5 | 23 | ✅ |
| 2 | 目录 | 4 | — | 4 | 5 | 5 | 18 | ⚠️ |
| ... | ... | ... | ... | ... | ... | ... | ... | ... |

### 需要修复

**Slide 3 — ⚠️ 图文协调 2 分**
- 问题：配图偏小（占面积<15%），被文字卡片包围，观众注意不到
- 修复：增大占位符区域从 180pt→300pt 宽，调整卡片布局给图留出空间
- 需要生图：否（仅调布局）

**Slide 6 — ❌ 图文协调 1 分 + 内容质量 2 分**
- 问题：配图是通用科技插画，与"GCN预测Tg精度"内容无关；数据裸奔无出处
- 修复：重写 Prompt 做 GCN 架构示意图；从 references/（知识库目录） 补充 Park 2022 引用
- 需要生图：是 → 新 Prompt: "Graph neural network architecture processing polymer repeat units into glass transition temperature predictions..."

### 生图清单（本次审查触发）

| 序号 | 对应页 | 文件名 | 新Prompt摘要 | 问题类型 |
|------|--------|--------|-------------|---------|
| 1 | 6 | p06_gnn_arch.png | GCN架构处理聚合物结构→Tg预测… | 内容不相关 |
```

---

## 6. 修复迭代上限

- 最多 **2 轮**审查+修复
- 第 1 轮：修复所有 ⚠️ 和 ❌
- 第 2 轮：仅修复仍未通过的页面（通常 < 3 页）
- 2 轮后仍不达标 → 标注 `[视觉降级]` 继续组装，不无限循环

---

## 7. 全局检查清单（所有页面 PASS 后，组装前）

- [ ] 相邻页无重复布局模式？
- [ ] 10页使用了 ≥5 种不同布局？
- [ ] 密度有呼吸感？（不连续 2 页同密度）
- [ ] 所有配图页 placeholder div 的 id 唯一且与新图片文件对应？
- [ ] 所有 text-only 页确实没有 placeholder div？
- [ ] IMAGE_MAP 中所有 key 对应的图片文件存在？
