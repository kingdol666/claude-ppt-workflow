## Study Style #20 双模学术系统（学术场景首选）

> **这不是一套"模板"，是一种创作方法论。** 融合了 Study Style 的人文质感与 SMG 学术报告的精密严谨，形成两个互补的视觉模式。

### 核心理念

每一页幻灯片根据其**论证目标**、**信息密度**和**叙事角色**，在暖学术模式和锐学术模式之间选择最合适的视觉语言。同一份 PPT 中两种模式可以共存——模式切换本身成为章节过渡的视觉信号。

### 暖学术模式（Warm Academic）— 默认基调

**灵感**：Nature/Science 信息图 + 大学出版传统 + TED 学术演讲的节奏感

| 元素 | 规范 |
|------|------|
| 页面底色 | `#FAF9F6` 暖纸白 — 温润、有质感、降低视觉疲劳 |
| 主色 | `#1B2A4A` 深蓝黑 — 标题、图表主色、节点、箭头 |
| 强调色 | `#C8963E` 暖琥珀金 — 关键数字、装饰短rule、强调节点 |
| 正文色 | `#1E1D1A` 暖近黑 — 段落文字 |
| 辅助色 | `#6B6760` 暖中灰 — 标签、脚注、来源标注 |
| 卡片底色 | `#F2F0EB` 暖浅灰 — 内容区块 |
| 重点卡片 | `#FEF9F0` 暖象牙白 + `#C8963E` 左边框 |
| 分割线 | `#E5E0D8` 暖浅褐 |
| 顶部边界 | `2pt solid #1B2A4A` 全宽细线 |
| 底部边界 | `1pt solid #E5E0D8` 全宽细线 |
| 页码 | `#C8963E` Georgia Bold 10pt, 右下角 |
| H1 标题 | 思源宋体/宋体 Bold, 22-30pt, `#1B2A4A` |
| H2 区块标题 | 微软雅黑 Bold, 13-16pt, `#1B2A4A` |
| Body 正文 | 微软雅黑 Regular, 10-12pt, `#1E1D1A` |
| 英文/数字 | Georgia Bold（标题/数字）/ Georgia Regular（正文） |

**适合页面**：封面、概念解释、总结收束、需要"呼吸感"和人文温度的页

### 锐学术模式（Crisp Academic）— 论证重音

**灵感**：学术答辩模板 + 期刊论文配图（Nature/Science/Cell 级别）

| 元素 | 规范 |
|------|------|
| 页面底色 | `#FFFFFF` 纯白 — 锐利、高对比、精密 |
| 主色 | `#000066` 海军蓝 — 正文、小标题、图表轴线 |
| 强调色 | `#A50021` 深绯红 — 标题、重点标注、关键数据 |
| 警示/重点 | `#C00000` 暗红 — 极重要结论 |
| 卡片底色 | `#F0F4F8` 浅蓝灰 — 内容区块 |
| 重点卡片 | `#FFF0F0` 浅绯红底 + `#A50021` 左边框 |
| 顶部标题栏 | `bg #A50021` h=36pt, `#FFFFFF` 文字 |
| 底部信息栏 | `bg #000066` h=24pt, `#FFFFFF` 文字 |
| 章节标签 | pill 形状, `bg #000066` 或 `bg #A50021`, `#FFFFFF` 白字, border-radius 3pt |
| H1 标题 | 微软雅黑 Bold, 24pt, `#A50021` |
| H2 区块标题 | 微软雅黑 Bold, 14pt, `#000066` |
| Body 正文 | 微软雅黑 Bold, 13-14pt, `#333333` |
| 英文/数字 | Times New Roman Bold / DIN Bold（数字标签） |

**适合页面**：数据密集页、方法对比、关键结论、"需要说服力"的论证页

### 模式切换指南

```
封面           → 暖学术（建立情感连接）
目录           → 暖学术（轻松导航）
背景/动机      → 锐学术（建立问题紧迫性，用数据冲击）
核心方法       → 暖学术（配图为主，文字为辅，呼吸感）
关键数据       → 锐学术（精确、权威、高对比）
应用案例       → 暖学术（多点并列，温润呈现）
挑战/瓶颈      → 锐学术（犀利指出问题）
总结           → 暖学术（温暖收束，留下印象）
```

**规则**：
- 同一章节内不切换模式（保持色彩连贯性）
- 模式切换本身成为视觉节奏的一部分——暖→锐→暖的交替制造"张弛有度"的阅读体验
- 封面和总结页必须使用暖学术模式

### 双模式生图 Prompt 风格关键词

**暖学术模式配图**（注入图片 prompt 末尾）：
```
clean academic schematic style, warm off-white #FAF9F6 background, 
navy blue #1B2A4A and amber gold #C8963E accents, 
flat vector illustration with subtle textures, 
no photographic elements, no gradients
```

**锐学术模式配图**（注入图片 prompt 末尾）：
```
publication-quality scientific figure, clean white background,
navy blue #000066 primary, crimson #A50021 accent,
crisp vector-quality lines, flat 2D style,
color-blind friendly, suitable for Nature/Science/Cell figures,
no rounded corners, no drop shadows, no gradients
```

### 配图内容锚点优先原则

> **Prompt 的前 60% 必须是内容描述，风格关键词只在末尾作为"滤镜"。**

正例：
```
"Silicon carbide crystal lattice showing tetrahedral SiC4 unit with sp3 bonding, 
alternating Si and C atoms, stacking sequence comparison of 3C vs 4H vs 6H polytypes, 
clean academic schematic, warm off-white background, navy+amber accents, flat vector, 4:3, no text"
```
↑ 内容锚点（SiC晶格、sp3键合、多型体堆垛）在前，风格关键词在后。

反例：
```
"A beautiful scientific illustration in navy blue and amber, academic style, 
showing some crystal structure, 4:3 ratio, warm background"
```
↑ 模板化、空洞——不知道要画什么具体的晶体结构。

## 风格参数获取流程

当用户选定风格后，按以下顺序获取完整参数：

1. **读 huashu-slides SKILL.md** → 确认风格编号和基础描述
2. **读 `$HS_DIR/references/proven-styles-gallery.md`** → 找到对应章节，提取：
   - CSS色值（HEX精确值）
   - 排版规则（字体族、字号、字重）
   - 构图规则（比例、网格、卡片样式）
   - Prompt要素（生图时注入的描述词汇）
3. **读 `$HS_DIR/references/prompt-templates.md`** → 获取 html2pptx 规则 + HTML模板

提取结果写入 PLAN.md "风格参数速查" 章节。

**特殊**：如果用户选择了 Study Style #20（学术场景首选），同时加载 SMG 学术报告 #19 的参数作为"锐学术模式"的补充色板。
