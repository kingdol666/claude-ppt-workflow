## 0. 配图决策规则（写逐页规划前必读）

> **最重要的原则：图片是论证工具，不是装饰品。内容决定是否需要图、需要什么样的图。**

### 决策流程

为每一页回答四个问题：

| # | 问题 | 判断标准 |
|---|------|---------|
| Q1 | **这页的核心信息是什么？** | 用一句完整的话写出——这是图片的"内容锚点" |
| Q2 | **纯文字能在3秒内传达这个信息吗？** | 能 → text-only；不能 → 继续Q3 |
| Q3 | **什么类型的视觉表达最适合这个信息？** | 结构示意 / 数据曲线 / 对比图表 / 流程示意 / 微观形貌？ |
| Q4 | **图和信息是互补关系（非重复）吗？** | 图仅重复文字 → text-only |

### 配图 Prompt 编写法则：内容锚点优先

```
Prompt = 内容锚点（60%）+ 风格修饰（25%）+ 布局约束（15%）
```

- **内容锚点**：具体描述要画什么——什么结构、什么数据关系、什么反应路径
- **风格修饰**：从 PLAN.md 风格参数速查表提取的风格关键词
- **布局约束**：比例和构图要求

反例（空洞的模板化 prompt）：
> "A futuristic technology illustration, blue tone, 16:9"

正例（内容驱动的 prompt）：
> "Silicon carbide crystal lattice showing tetrahedral SiC4 unit with sp3 bonding at 109.5°, alternating Si (blue) and C (silver) atoms, stacking comparison of 3C(ABC) vs 4H(ABCB) polytypes, clean academic schematic, warm off-white background, navy+amber accents, flat vector, 4:3, no text"

### 图片类型与内容匹配

| 页面要传达什么 | 最适合的图片类型 | 示例内容锚点 |
|-------------|---------------|------------|
| 晶体结构/分子构型 | 三维结构示意图 | "tetrahedral SiC4 unit, sp3 bonds at 109.5°" |
| 实验数据趋势 | 数据曲线图 | "TGA weight gain curve, three distinct thermal regions" |
| 反应路径/机理 | 流程示意图 | "Si+C → a-Si(C) amorphous intermediate → β-SiC crystallization" |
| 性能对比 | 对比图表 | "band gap comparison bars: Si 1.1, 3C-SiC 2.36, 4H-SiC 3.2 eV" |
| 设备/装置原理 | 装置概念图 | "fluidized bed CVD reactor with precursor inlet and heating zones" |
| 微观形貌 | 显微图像示意 | "monodisperse spherical nanoparticles, diameter 5-100 nm" |
| 纯文字论证 | **text-only** — 文字本身足够有力 | — |
| 封面 | 1张 hero-cover — 建立视觉身份 | 项目核心研究对象的艺术化科学呈现 |

## 内容要点编写原则

> **PLAN.md 的"内容要点"是创作指南，不是幻灯片文案。**

每个要点应包含：
- **论点方向**：这一个卡片/区块要论证什么观点？
- **数据锚点**：关键数字+引用来源（供下游扩展时引用）
- **扩展提示**：这个点需要什么样的背景解释？（可选，复杂概念时提供）
- **密度期望**：这页整体是极简/中等/深度？（决定下游展开程度）

**密度决定要点的写法**：
- **极简页**要点：给下游"最核心的一个数字/一句话"——其他全部删掉
- **中等页**要点：给下游"核心论点+关键数据+1个引用"——80字以内
- **深度页**要点：给下游"完整论证链+多个数据+2-3个引用"——充分展开

**反例**（写成了幻灯片文案）：
```
要点1: GCN 预测 Tg: R² ≈ 0.9 (Park 2022)
→ 下游只能复制这 10 个字到幻灯片上
```

**正例**（写成了创作指南）：
```
要点1: GCN预测Tg精度——R²≈0.9，>10k数据训练，预测误差±15°C已接近实验离散范围
       (Park 2022 ACS Polymers Au) — [密度：深度，需展开解释GCN方法和R²含义]
→ 下游知道：核心论点是"精度接近实验可重复性"，数据锚点是 R²=0.9/10k/±15°C，
   需要扩展解释 GCN 方法和 R² 含义
```

**极简页正例**：
```
要点1: R² = 0.90 — GCN预测Tg的确定性系数 — [密度：极简，仅此一个数字+标签]
→ 下游知道：这是数据冲击页，只放这个数字，不做任何展开
```

**关键规则**：要点文字不是幻灯片上要显示的文字。你是在给下游的"内容创作者"写指引。**同时通过密度标注告诉下游：这页该展开多少。**

## 布局创作原则：每页独立设计

> **PLAN.md 中的 `布局` 字段不再是"从库中选择一个名字"，而是对该页排版方案的原创描述。**

### 核心理念：内容驱动布局

**设计的起点是论证逻辑，不是视觉模板。** 为每一页回答：
1. 这页要论证什么？（论点）
2. 观众需要在3秒内看到什么？（视觉主角）
3. 什么空间组织方式最能传达这个论点？（排版方案）

然后根据答案**创作**该页的排版描述——可以从 huashu-slides 的 `references/proven-styles-gallery.md` 中对应风格的"布局模式库"部分获取灵感，但必须根据该页具体内容进行定制。

### 硬性创作约束

1. **相邻页不得使用相同的空间组织方式。** 封面和总结页也不例外。
2. **10页至少使用6种不同的排版逻辑。** "不同"是指真正不同的空间组织——不是把2×2网格改成2×3就算不同。
3. **禁止连续3页出现相同类型的组件**（如连续3页都用 `border-left` 卡片）。
4. **每页有且仅有一个视觉主角**——图、超大数字、对比结构、或核心论点卡片。
5. **留白是设计元素**——极简页留白≥40%，深度页留白≥20%。

### 创意排版技法（鼓励使用，打破千篇一律）

- **非对称分栏**：不总是 50:50。尝试 65:35、黄金比例 62:38、或 70:30
- **重叠侵入**：关键元素可以略微侵入相邻区域，通过负 margin 制造层次
- **负空间聚焦**：大量留白包围一个小而重要的元素，注意力反而更集中
- **尺度对比**：同一页内 120pt 数字 vs 9pt 脚注——尺度差就是信息层级
- **色彩重音**：在暖学术页面序列中，第4页切换到锐学术模式的关键数据页
- **不规则网格**：不总是等大等距的卡片——让内容重要性决定卡片大小

### 每页规划中的 `布局` 字段格式（新版）

不再使用固定模式名称。改为对该页排版方案的原创描述：

```
- **布局**: 非对称双栏（左65%右35%），左栏垂直4卡片叠放（每卡片border-left琥珀色3pt），右栏居中1张占位图（270pt×210pt）+图注，章节指示右上角，密度：中等
- **布局**: 全宽三列非等宽（3:4:3比例），中列120pt超大数字Georgia Bold #C8963E居中+下方琥珀短rule 48pt，两侧列各2条9pt要点，密度：极简
- **布局**: 上下分区——上半全宽占位图（680pt×110pt带隙对比），下半左右分栏（各50%），每栏标题+正文卡片，密度：中等
- **布局**: 2×2不规则网格——左上大卡片（2倍高，核心概念）+右上小卡片+左下小卡片+右下占位图（280pt×140pt），密度：中等
- **布局**: 顶部断言句标题22pt宋体居中+琥珀短rule 48pt+下方5节点水平递进（节点圆40pt深蓝底白字+箭头1pt琥珀+每节点下方一行8pt标签），密度：极简
```

### 密度选择与视觉节奏

| 因素 | 倾向极简（10-40字） | 倾向深度（80-150字） |
|------|-------------------|---------------------|
| 观众背景知识 | 已有 → 视觉提示即可 | 陌生 → 需要展开解释 |
| 页面的叙事角色 | 冲击/过渡/收束 | 论证/教育/说服 |
| 信息可查性 | 能（众所周知的事实） | 不能（独家数据/分析） |

**节奏规律：密→疏→密→疏交替，不连续2页同为深度页。**

## 六、知识直通

> **PLAN.md 是骨架，`references/` 是弹药库。** 下游技能 huashu-slides 必须同时拿到两者才能创作出有深度的幻灯片。

- PLAN.md 的"内容要点"是创作指引（含数据锚点和扩展方向），不是最终幻灯片文字
- 幻灯片构建者（huashu-slides）应从对应 `ref-*.md` 提取具体数据、引用、背景解释，而非仅依赖 PLAN.md 中浓缩后的要点

## 字段约束

### 标题（断言句原则）
- ✓ "GNN突破材料性能预测精度上限，误差<0.05 eV/atom"
- ✓ "生成模型实现材料逆向设计，已发现10+新型功能材料"
- ✗ "材料性能预测"
- ✗ "深度学习简介"

### 内容要点（信息密度原则）
- ✓ "从10^6候选钙钛矿中筛选出10个最优催化剂，速度提升1000倍"
- ✓ "主动学习+贝叶斯优化将DFT计算量降低10-100倍"
- ✗ "深度学习可以预测材料性能"（空洞无数据）
- ✗ "人工智能是重要工具"（废话）

### 图片决策（内容锚点优先原则）
编写逐页规划时，每一页必须回答：**这页的核心信息是什么？纯文字能在3秒内传达吗？最适合的视觉表达是什么？**
- 图片的 Prompt 必须以具体的内容锚点开头（这页要展示什么结构/数据/概念？）
- 风格关键词只作为 Prompt 末尾的"滤镜"，不占据前半部分
- 目录/总结/致谢页 → 默认 text-only
- 封面 → 默认 1张 hero-cover（附具体的内容锚点——研究对象是什么？）
- 当不确定时 → 问自己"去掉图信息会受损吗？"，不会 → text-only

### 配图Prompt（三要素原则）
- ✓ "Graph neural network transforming crystal structure into band gap predictions, clean navy+crimson academic schematic with labeled nodes, flat vector style on white background, 4:3, no text"
  （内容锚点明确 + 风格关键词 + 布局约束）
- ✗ 空洞通用："A futuristic technology illustration, blue tone, 16:9"
  （无内容锚点、无风格关键词）
- ✗ 超过5句话，指定字号、颜色比例、构图位置
- ✗ 不包含风格关键词（生成结果与PPT整体风格割裂）

### 配图文件名规范
- 格式：`p<页码两位数>_<内容关键词>.png`
- 示例：`p01_cover.png`, `p03_prediction.png`, `p06_screening.png`

### 占位符 id 稳定合约
- **占位符 id 是稳定合约**：图片文件可以重新生成（审查后换图），但 HTML 中的占位符 id 保持不变
- 换图时：新图覆盖旧文件（同名同路径），或更新 `IMAGE_MAP` 中该 id 映射到新文件名
- 改图尺寸时：更新 HTML 占位符 CSS 区域 + 用新比例重新生图
- 增图时：HTML 添加新占位符（新 id）+ PLAN.md 配图清单追加条目
- 删图时：HTML 移除占位符 + PLAN.md 标记为 text-only

## 风格参数来源

PLAN.md中的"风格参数速查"必须从以下来源精确提取：

1. 选定风格 → 读 `$HS_DIR/references/proven-styles-gallery.md` 对应章节
2. 提取 CSS 色值（HEX）、排版参数、构图规则、**Prompt要素**（生图风格关键词）
3. 写入 PLAN.md 供 huashu-slides 直接使用，无需再查文档
4. **生图风格关键词**单独列出，供 gpt-image-2 的配图Prompt注入使用

**目的**：huashu-slides 拿到 PLAN.md 后可以直接开始制作HTML，无需回头查询风格参数；gpt-image-2 拿到配图Prompt后可直接生成与PPT风格一致的图片。
